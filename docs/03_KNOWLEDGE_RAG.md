# 📚 قاعدة المعرفة والتمرّس (RAG + التعلّم)

> هذا هو الجزء الذي يجعل الوكيل "يتمرّس" ويصبح خبيراً.

## الفكرة
الوكيل لا يصبح أذكى بتغيير دماغه — يصبح **خبيراً** بتراكم المعرفة:
1. **RAG** — معرفة تقنية جاهزة (توثيق React Native/Next.js/Angular)
2. **التعلّم** — خبرات يكتسبها من كل مهمة ناجحة

كلاهما يُحقن في عقله **قبل** كل مهمة، فيبني بدقة مستنداً لمعرفة حقيقية.

---

## 🔍 كيف يعمل RAG

```
مهمة جديدة: "ابنِ صفحة Next.js"
       ↓
knowledge.context_for(task, topic="nextjs")
       ↓
يبحث في 85 مقطع توثيق (SQLite FTS5)
       ↓
يسترجع أكثر 4 مقاطع صلة
       ↓
يحقنها في رسالة الوكيل قبل التنفيذ
       ↓
الوكيل يبني مستنداً للتوثيق الرسمي الفعلي ✅
```

التقنية: **SQLite FTS5** — بحث نصّي كامل، سريع، بدون تبعيات ثقيلة (لا حاجة GPU/embeddings).

---

## 🧠 حلقة التعلّم

بعد كل مهمة ناجحة:
```
المهمة نجحت → done:true
       ↓
العقل يستخلص "خبرة" (name + approach + tags)
       ↓
تُحفظ في جدول skills
       ↓
المرة القادمة: تُسترجع كـ "خبرة سابقة" وتُحقن
       ↓
الوكيل أسرع وأدق (يتذكّر كيف حلّها سابقاً) ✅
```

## 📂 الملفات

| الملف | الوظيفة |
|-------|---------|
| `knowledge/knowledge.py` | المحرّك (تخزين + بحث RAG + skills) |
| `knowledge/ingest.py` | تغذية التوثيق من المصادر الرسمية |
| `knowledge/index/knowledge.db` | قاعدة البيانات (FTS5) |

## 🛠️ الاستخدام

### إضافة توثيق تقنية جديدة
عدّل `SOURCES` في `ingest.py` (أضف روابط raw markdown)، ثم:
```bash
python3 knowledge/ingest.py
```

### فحص المعرفة
```bash
python3 -c "import sys;sys.path.insert(0,'knowledge');from knowledge import Knowledge;print(Knowledge().stats())"
```

### تشغيل وكيل متمرّس على تقنية
```bash
python3 agent/agent.py --topic nextjs "ابنِ لي صفحة هبوط"
# أو في الواجهة التفاعلية:  /topic nextjs
```

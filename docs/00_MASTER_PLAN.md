## 🏛️ المعمارية (كيف بُني؟)

```
┌──────────────────────────────────────────────────┐
│                  Ai_Alabbasi                      │
│                                                   │
│  👤 المستخدم (عبّاس)                               │
│        ↓ يعطي مهمة                                │
│  ┌────────────────────────────────────────────┐  │
│  │  🔁 AGENT LOOP (حلقة الوكيل)                 │  │
│  │     يفكّر → يختار أداة → ينفّذ → يراقب → كرّر  │  │
│  └────────────────────────────────────────────┘  │
│        ↓ يطلب تفكير              ↓ يطلب تنفيذ      │
│  ┌──────────────┐         ┌────────────────────┐  │
│  │ 🧠 BRAIN      │         │ 🛠️ TOOLS           │  │
│  │ (عقل موحّد)   │         │ - تيرمنال          │  │
│  │ يبدّل النموذج │         │ - ملفات (قراءة/كتابة)│  │
│  │ من ملف واحد  │         │ - تشغيل أوامر       │  │
│  └──────────────┘         │ - بحث ويب          │  │
│        ↓                  └────────────────────┘  │
│  ┌──────────────────────────────────────────┐    │
│  │ 📚 KNOWLEDGE (التمرّس)                      │    │
│  │ - توثيق التقنيات (RAG)                      │    │
│  │ - skills متراكمة من التجارب                 │    │
│  └──────────────────────────────────────────┘    │
└──────────────────────────────────────────────────┘
```

### مبدأ التصميم الذهبي:
> **العقل معزول في طبقة واحدة (`brain/`).** أي تبديل للنموذج = تعديل `active_brain` في `brain_config.yaml` فقط. لا يُلمس أي ملف آخر. هذا يحقق طلب عبّاس: "حتى لو نبي نربطه بنموذج محلي نقدر من نفس الصفحة."

|--------|---------|

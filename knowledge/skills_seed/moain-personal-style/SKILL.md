# 👤 أسلوب معين الشخصي

مهارة تلتقط تفضيلات معين (المالك) لتجعل عمل الوكيل متوافقاً مع توقّعاته.

## متى تُستخدم
- في كل مهمة (مهارة سياقية عامة).
- خاصة عند بناء واجهات أو كتابة كود أو شرح خطوات.

## التفضيلات الأساسية
1. **الدقة والقوة**: "ابدأ بدقة وقوة" — نفّذ بإتقان لا باستعجال.
2. **العربية**: التواصل والتوثيق والكود (أسماء الدوال/المتغيّرات) بالعربية حيث ممكن.
3. **خطوات واضحة**: اشرح ما ستفعله، نفّذه، ثم أكّد النتيجة.
4. **اختبار بعد كل تغيير**: لا تعلن النجاح قبل التحقّق الفعلي بالتشغيل.
5. **لا توقّف**: عند منح الصلاحية، أكمل المهام تباعاً دون انتظار غير ضروري.
6. **الأمان أولاً**: احترم حدود `_safe_path` والمفاتيح في متغيّرات البيئة.
7. **التزامات Git**: commit ذرّي بعد كل تغيير + PR، لا تلمس main مباشرة.

## القواعد الذهبية (تُطبّق دائماً)
- **العقل القابل للتبديل** (Swappable Brain): النموذج يُغيَّر من ملف واحد.
- **أقل اعتماديات**: فضّل stdlib؛ أي تبعية ثقيلة تكون اختيارية مع سقوط آمن.
- **commit ذرّي**: تغيير واحد مركّز لكل التزام برسالة واضحة.

## نبرة التواصل
- مباشرة، واثقة، محترمة، بالعربية.
- استخدم الرموز التعبيرية باعتدال للوضوح (✅ ⚠️ 🔧).
- لخّص الإنجاز في نهاية كل مهمة كبيرة.

## مثال على التطبيق
عند طلب ميزة جديدة:
1. اشرح الخطة بإيجاز.
2. نفّذها بدقة (كود عربي نظيف).
3. اختبرها فعلياً.
4. commit + PR.
5. لخّص: ما أُنجز + الرابط + المتبقّي.

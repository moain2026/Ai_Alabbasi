## 🔍 1. Hybrid Search (FTS5 + Vector + RRF)

**السؤال:** كيف ندمج البحث الدلالي مع بحث الكلمات المفتاحية بأقل تكلفة؟

**الاستخلاصات:**
- **sentence-transformers** يعمل **محلياً بالكامل بدون API** — نموذج مثل `all-MiniLM-L6-v2`
  (~80 MB، 384-بُعد) سريع وكافٍ للوثائق التقنية.
- **RRF (Reciprocal Rank Fusion):** صيغة بسيطة لدمج نتائج بحثين بترتيب موحّد:
  `score(d) = Σ 1/(k + rank_i(d))` حيث `k≈60`. لا تحتاج معايرة أوزان.
- **الأداء المرصود:** الدمج (FTS5 + vectors عبر RRF) يحقّق **~92.3% Recall@5**، ويتفوّق على
  البحث الدلالي وحده وعلى الكلمات المفتاحية وحدها.
- **التخزين:** يمكن تخزين الـ embeddings في عمود BLOB في SQLite بجانب جدول FTS5 الحالي
  (أو امتداد `sqlite-vec`) — يحافظ على فلسفة "ملف واحد، خفيف".

**كيف نطبّقه:** `knowledge/embedder.py` (يولّد embeddings) + `knowledge/hybrid_search.py`
(يجمع FTS5 + vectors عبر RRF). انظر ADR-001.

## 🔍 2. Claude Agent Skills — Progressive Disclosure

**السؤال:** كيف ندمج 717 مهارة دون إغراق السياق؟

**الاستخلاصات (معيار Anthropic المفتوح، ديسمبر 2025):**
- **SKILL.md** = YAML frontmatter (`name`, `description`) + Markdown body.
- **3 طبقات كشف تدريجي:**
  1. **Discovery** (~80 token/مهارة): الاسم + الوصف فقط، تُحمَّل دائماً.
  2. **Activation** (~2000 token وسيط): الجسم الكامل، يُحمَّل عند اختيار المهارة.
  3. **Execution** (عند الطلب): scripts/references، تُقرأ فقط عند الحاجة.
- الفائدة: 717 مهارة × 80 token = ~57K token للاكتشاف، لكن نحمّل الجسم الكامل لمهارة واحدة فقط
  في كل مرة → توفير هائل.

**كيف نطبّقه:** فهرسة الطبقة 1 فقط في `docs`، وتحميل الطبقة 2 عند الاختيار. انظر `03_تحليل_المهارات.md §5`.

## 🔍 3. CodeAct vs Function Calling

**السؤال:** هل CodeAct يستحق الإضافة؟

**الاستخلاصات:**
- **CodeAct** = الوكيل يكتب كود Python قابل للتنفيذ كـ "فعل" موحّد، بدل اختيار دالة من قائمة ثابتة.
- المزايا: تركيب أدوات متعددة في خطوة واحدة (loops, conditionals)، مرونة أعلى، خطوات أقل.
- المخاطر: تنفيذ كود عشوائي → **يجب أن يبقى داخل `_safe_path` / sandbox**.
- التطبيق الآمن: `code_act_executor.py` ينفّذ في بيئة محصورة، يلتقط stdout/stderr كـ Observation.

**القرار:** نضيف CodeAct كأداة **إضافية** بجانب الأدوات الـ7 (لا نستبدلها). انظر ADR-002.

## 🔍 4. Daytona / E2B Cloud Sandbox

**السؤال:** كيف ننقل التنفيذ للسحابة مع توفير التكلفة؟

**الاستخلاصات:**
- **Daytona SDK** (Python): إنشاء sandbox سحابي معزول لكل مهمة، مع **Hibernation** (نوم عند الخمول = 0$).
- **E2B**: بديل مشابه، sandboxes سريعة الإقلاع.
- **Zero-Trust isolation:** كل مهمة في حاوية منفصلة → أمان أعلى من المحلي.
- التصميم: `tools/cloud_sandbox.py` كـ backend اختياري؛ الافتراضي يبقى `local` (احترام الخفّة).

**القرار:** backend قابل للتبديل (`local`/`daytona`/`e2b`) عبر config. انظr ADR-004.

## 🔍 5. KV-Cache Optimization (درس Manus الأول)

**الاستخلاصات:**
- نسبة الإدخال/الإخراج في الوكلاء ~**100:1** → الـ prompt الطويل يهيمن على التكلفة.
- الـ tokens المخزّنة (cached) **أرخص ~10×**.
- شروط الاستفادة: **بادئة ثابتة** (system prompt لا يتغيّر) + **append-only** + **JSON حتمي** (`sort_keys=True`).
- **تطبيق فوري رخيص:** إضافة `sort_keys=True` في `json.dumps` داخل `agent.py`.

**القرار:** تحسين سريع في المرحلة 1. انظر ADR-003.

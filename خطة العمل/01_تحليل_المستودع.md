## 🏗️ 1. تشريح الملفات (ملفاً ملفاً)

### 🚪 `run.py` — نقطة الدخول
- واجهة CLI تفاعلية مع slash commands: `/brain` (تبديل النموذج)، `/topic`، `/kb` (المعرفة)، `/skills`.
- يستورد `Agent` من `agent/` و `Knowledge` من `knowledge/`.
- دوال: `list_brains()`، `set_brain()`، حلقة `main()` مع `input("👤 عبّاس> ")`.
- **التقييم:** نظيف وواضح. نقطة دخول مثالية لإضافة أوامر جديدة (مثل `/plan`، `/interrupt`).

### 🧠 `brain/brain.py` — العقل الموحّد ⭐
- فئة `Brain` = نقطة الاتصال الوحيدة بأي LLM.
- `__init__` يحمّل الإعدادات من YAML.
- `think(messages, temperature, max_tokens)` ينفّذ HTTP POST إلى `{base_url}/chat/completions` عبر **urllib** (بدون مكتبات خارجية) مع إعادة محاولة عند 429.
- `_api_key()` يقرأ من متغيرات البيئة (أمان جيد).
- يدعم مزوّدي: `openrouter` / `genspark` / `openai_compatible`.
- **التقييم:** عبقري في بساطته. **يجب عدم ربط أي كود جديد بنموذج واحد** — كل شيء يمرّ عبر `think()`.

### 🎛️ `brain/brain_config.yaml`
- `active_brain: genspark_claude_opus`.
- 8 أدمغة معرّفة.
- `defaults: temperature 0.3, max_tokens 4096, timeout 120, retry_attempts 3`.

### 🔁 `agent/agent.py` — حلقة ReAct (9.6 KB)
- فئة `Agent` بحلقة ReAct.
- `SYSTEM_PROMPT` يفرض إخراج JSON فقط.
- `extract_json(text)` محلّل JSON.
- `Agent.__init__(max_steps=25, verbose, topic, learn, on_event)`.
- `run(task)` الحلقة الرئيسية.
- `_save_skill()` للتعلّم الذاتي.
- `_emit(kind, data)` لبث الأحداث.
- يكتشف أخطاء المزوّد ليتوقف مبكراً.
- ⚠️ **ملاحظة تحسين:** يستخدم `json.dumps(action, ensure_ascii=False)` **بدون `sort_keys`** → يكسر استقرار KV-cache (انظر ADR-003).

### 🛠️ `tools/tools.py` — 7 أدوات آمنة
- `PROJECTS_ROOT` = حدود الـ sandbox.
- `_safe_path(path)` = حارس الأمان (يمنع الخروج من `projects/`).
- الأدوات: `run_shell` · `write_file` · `read_file` · `append_file` · `list_dir` · `make_dir` · `web_search` (DuckDuckGo).
- سجلّ `TOOLS` dict + `tools_spec()` + `call_tool(name, args)`.
- **التقييم:** نمط تسجيل أدوات نظيف وقابل للتوسعة. سنضيف هنا: `file_str_replace`، جلسات shell، CodeAct.

### 📚 `knowledge/knowledge.py` — RAG + التعلّم الذاتي
- فئة `Knowledge` فوق SQLite FTS5.
- جدول `docs` (topic, title, content, source) + جدول `skills` (name, task, approach, code, tags, created).
- دوال: `add_doc`، `search_docs`، `add_skill`، `search_skills`، `stats`، `context_for(task, topic)` (لحقن RAG).
- `_fts_query()` يبني استعلام OR.
- **التقييم:** البنية ممتازة للتوسعة نحو Hybrid Search (إضافة عمود/جدول vectors بجانب FTS5).

### 📥 `knowledge/ingest.py` — التغذية التلقائية
- `SOURCES` dict (react-native، nextjs، angular).
- دوال: `fetch`، `clean`، `chunk(text, size=1500)`، `main()`.
- **التقييم:** يحتاج توسعة من 3 تقنيات إلى 50+.

### 🌐 `web/server.py` — FastAPI + SSE
- `load_env()`، قاعدة محادثات SQLite.
- `/api/chat/stream` مع `on_event` callback يدفع للقائمة.
- `/api/brains`، `/api/conversations`.
- uvicorn على المنفذ 8000.
- **التقييم:** يحتاج ترقية إلى WebSocket لبث الأحداث الحيّة ودعم المقاطعة.

### 🔑 `config/.env.example`
- `GENSPARK_API_KEY`، `OPENROUTER_API_KEY`، `DUMMY_LOCAL_KEY`.

### 📖 `docs/02_STATUS.md`
- يؤكد 85 مقطع مفهرس، وأن الوكيل أصلح بنفسه خطأ `python→python3`، وأن RAG يعمل.
